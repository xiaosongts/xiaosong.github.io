代理模式

  • 思想：找个中间商，在自己能干的事情基础上干自己不能干的事情，可以用来保护自己
  • 进行代码易维护、进行功能增强
  • 结构
    • 目标对象
    • 代理对象
    • 目标对象和代理对象的公共接口或者继承
  • 静态代理：has-a关系 关联关系
  • 动态代理
    • JDK内置的java.lang.reflect.Proxy 只能代理接口
    • cglib 底层采用继承，可以代理接口或者类 底层使用ASM字节码处理框架优化
    • javassist（千叶滋yyds）

AOP(一个思想，面向切面编程)

  • Aspect Oriented Programming
  • 思想：切面是一个和业务逻辑毫无关系的东西；采用代理模式，把这些切面当做功能增强的东西单独提取出来，形成一个单独的组件，以横向交叉的方式应用到业务流程。
  • 作用：代码复用、可读性高
  • 交叉业务

Spring AOP

  • 采用 JDK代理+cglib代理

术语

名称 意义
JoinPoint 连接点 切面可以切入的位置
CutPoint 切点 方法，被切的东西
Adivce 通知 具体增强的代码，切东西的工具
Aspect 切面 切点+通知
Weaving 织入 具体的动作
Target 目标对象
Proxy 代理对象

通知的类型和顺序

  • 前置通知
  • 环绕通知
  • 后置通知
  • 异常通知(当调用此通知，后续通知不再执行，若异常目标方法自己处理了，后续通知会继续执行)
  • 返回通知
try{
	·前置通知
	·前环绕通知
	·目标方法
	·后环绕通知
	·后置通知
	·返回通知
}catch(Exception e){
	·异常通知
}

切点表达式

实例

  • public class Student {
    private String name;

    public Object exam(){
        System.out.println("目标方法  切点...." + name + "学生正在考试！");
        int a=1/0;//自己捕获异常，处理的话，切面不处理了
        return name;
    }
    public void end(){
        System.out.println("目标方法  切点...."+name+"学生陆续交试卷走人！");
    }
    public void setName(String name) {
        this.name = name;
    }
}

  • public class ProctorAspect {
    public void tellStartTime(){
        System.out.println("监考官提醒时间开始！");
    }
    public void tellStart(){
        System.out.println("监考官提醒开始考试！");
    }
    public void invigilate(ProceedingJoinPoint proceedingJoinPoint) throws Throwable {
        System.out.println("前环绕....监考官正在监考！");
        proceedingJoinPoint.proceed();
        System.out.println("后环绕....监考官正在监考！");
    }
    public void tellEndTime(){
        System.out.println("监考官提醒时间还有五分钟！");
    }
    public void tellEnd(){
        System.out.println("监考官开始收卷！");
    }
    public void exceptionHHH(Exception e ){
        System.out.println("停电！");
    }
    public void after_return(){
        System.out.println("考试方法....你return了？");
    }
}

注解方式

<aop:aspectj-autoproxy/>最为重要（若基于类则使用 @EnableAspectJAutoProxy）

  • 声明切面的注解

@Aspect

  • 声名切点的注解

@PointCut

  • 通知的注解

@Before

@Around

@After

@AfterThrowing

@AfterReturning

XML配置方式

  • <!--目标对象-->
<bean id="student" class="com.yushan.pojo.Student">
    <property name="name" value="lisi"/>
</bean>
<!--代理对象-->
<bean id="proctorAspect" class="com.yushan.pojo.ProctorAspect"/>

<aop:config>
    <!--配置切点表达式-->
    <aop:pointcut id="exam" expression="execution(* com.yushan.pojo.Student.exam(..))"/>
    <aop:pointcut id="end" expression="execution(* com.yushan.pojo.Student.end(..))"/>
    <!--配置切面-->
    <aop:aspect ref="proctorAspect">
        <!--前置通知-->
        <aop:before method="tellStartTime" pointcut-ref="exam"/>
        <!--前置通知-->
        <aop:before method="tellStart" pointcut-ref="exam"/>
        <!--环绕通知-->
        <aop:around method="invigilate" pointcut-ref="exam"/>
        <!--前置通知-->
        <aop:before method="tellEndTime" pointcut-ref="end"/>
        <!--后置通知-->
        <aop:after method="tellEnd" pointcut-ref="end"/>
        <!--异常通知-->
        <aop:after-throwing method="exceptionHHH" throwing="e" pointcut-ref="exam"/>
        <!--返回通知-->
        <aop:after-returning method="after_return" pointcut-ref="exam"/>
    </aop:aspect>
</aop:config>
